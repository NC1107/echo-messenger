## CRITICAL (block beta)

### C1. LiveKit voice token IDOR — issue token for arbitrary room
**Source**: security-expert #1 — VERIFIED via Read of voice.rs:80-130.
**File**: apps/server/src/routes/voice.rs:86-130
**Evidence**: `room` is taken from `body.room.or(body.channel_id).or(body.conversation_id)` but the membership check (line 117) runs against `body.conversation_id.or(body.channel_id)`. Attacker-as-member-of-A sends `{room: "victim-room", conversation_id: "my-conv"}` and gets a LiveKit token with `roomJoin/canPublish/canSubscribe` on the victim's room.
**Fix**: derive `room` from the validated conversation_id/channel_id only. Drop `body.room` from the request type, or require `room == conversation_id`.
**Effort**: small
